學習心得 | SAS Advanced Programming 2022/5/16

  2022-05-16   學習心得   SAS 0

考前程度

使用SAS處理資料庫有7年的時間，常用MACRO和SQL，但完全沒碰過hash object和FCMP function

考試規定

  • 總分：200~1000，725及格
  • 考試時間：2h5m (total:2h30m)
  • 考試代碼：A00-232
  • 費用：原價180USD，教育折扣後75USD (用學校信箱註冊SAS Skill Builder取得折扣碼)

備考策略

  1. 找到63題+20新題，和LAB機經(by Mike Q)的PDF (我找超久，這邊直接附上來~題號前每個顏色點點代表有考過的人，粉色勾勾代表我這次有考到，連結不定期失效)
    • 63題+20新題 https://reurl.cc/vda7jy
    • LAB機經 https://reurl.cc/0pv18x
  2. 把前人分享的文章(連結附在最下面)統一整理到OneNote，並把有考過的題號標記在PDF上(可以發現會考的就那幾題XD 沒考過的我就沒看了)，熟讀這些題目並理解九成以上(我還是不懂dictionary.columns做啥用的)
    • 時間愈近的文章實際考題命中率愈高，資源大多來自一畝三分地
  3. 非常建議做practice exam，熟悉考試模式，稍微檢視一下還有哪些概念不熟悉
  4. 準備時間：前面準備比較零碎，一部份63+20題、LAB機經、考前準備和practice exam集中在最後三天，全部加起來大約一個多禮拜 (花蠻多時間看hash和except/intersect/union/outer union corr/all)

考前準備

  1. 官網的線上考試操作教學要看，可以替上機考省下很多時間 https://www.sas.com/zh_tw/certification/credentials/foundation-tools/advanced-programming-professional.html
  2. 一定要先測試考試軟體是否能在電腦上正常運作，選填考SAS advanced時或報考後的email都有提供測試連結，測試內容包含麥克風、喇叭、視訊鏡頭和wifi網路是否正常運作(這邊注意不能連手機熱點，不然proctor可能會看不到你，導致測試不通過)，另外如果它有發現電腦正在運行遠端應用程式(例如spacedesk)會提示要你關掉
  3. 線上考試有很多時段，我是準備得差不多，考前十九小時才報名
  4. 複習考古題(LAB機經要背熟！)、practice exam和前人文章
  5. 準備護照、清空桌面。建議著裝整齊，因為你的成績單上會有你的自拍照

OnVUE online proctored exam

  1. 登錄流程：電腦登入 → 手機驗證身分(自拍、拍證件、拍座位前後左右) → 回到電腦等待proctor檢查 → proctor A打來叫我用電腦鏡頭拍桌面(proctor A口音超重，一直聽不懂他要我幹嘛，請他打在chat box才懂) → 自動進入考試軟體
    • 驗證身分方式可以選要QR Code連手機網頁、電腦或app，覺得手機拍照比較方便就用QR Code
  2. Introduction → 開始計時考試
    • 我要進到introduction前有短暫斷線，proctor B馬上在對話框叫我趕快按照指示退出重新登入，否則很有可能考試無效而且不能退費，我讀指示讀比較久，他一直傳訊息催促，我整個很慌OAO 還好後來換一個wifi重連後就沒問題了
  3. 考題
    • LAB：1、2、3、5、6、8、10、12、15、17、20
      • 評分方式：幾乎所有LAB題都是執行cert/program/score.sas裡的%scoreit(你寫的program題號.sas)，填入回傳的隨機三位數數字為唯一解，所以你也無法知道它到底是不是正確答案，而其中只有一題不是(用mprint看log，問parameter填N，問variable填age)
    • 選擇及填空：3、4、9、12、28、36、38變、44變(選有dictionary.columns那個)、49、55、58、60、新2、新8、新12、perl regular expression的email地址格式(有.和@和(com|org|edu))、lag2(x)第四個iteration是啥

心得和後記

  • 考試寫SAS的介面很lag，一開始大於、小於還打不出來，寫幾題之後突然又正常了(可能跟網速也有關係)
  • SQL case when那題我想說之前常用，在準備時就直接略過，結果考試寫 case when x<25000 then ‘L’ case when x>45000 then ‘H’一直ERROR跑不出來就直接交了好可惜😅
  • 考到一個半小時時電腦快沒電趕快接充電線，但因為換了一個充電線不確定到底有沒有充到電，做完一遍沒檢查就趕快交卷了，最後876過
  • 取之於前人，用之於後人，希望這篇文能幫助正在備考同學，加油！
